01 · Resumen ejecutivo

Qué encontramos en una frase

Coquimbo Unido tiene una base pequeña pero muy limpia y profundamente local: casi todos son de Coquimbo, casi todos tienen teléfono y correo válidos, y el RUN valida en el 97%. El trabajo real no es "arreglar datos malos" — es depurar duplicados, enriquecer y activar a una comunidad que ya está lista para ser contactada.

¿Podemos confiar en estos datos?

Antes de analizar nada, revisamos si la base sirve. Estos son los problemas reales que detectamos y cómo los resolvió el pipeline de limpieza.

Problema 1 · Edad vacía

La columna "Edad" viene vacía en el 100% de los 325 registros. No sirve como dato base. La reemplazamos por una estimación honesta desde el RUN (ver sección Edad).

Problema 2 · Teléfonos dañados

Excel convirtió móviles a notación científica (5.69e+10). El script los reconstruye al formato chileno +569XXXXXXXX. Resultado: 312 móviles válidos.

Problema 3 · Ciudad con typos

"COQUIMBI", "COQUIMNO" y variantes son errores de tipeo de Coquimbo. Se normalizaron para no fragmentar el mapa territorial.

Veredicto: la base es apta para análisis y activación. Calidad alta en 260 registros, 47 requieren validación (duplicados) y solo 10 son de baja calidad. Completitud promedio: 99,9%.

Qué nos dice el perfil

La base está dominada por dueñas de casa (29) y estudiantes (16), seguidas de oficios y profesiones variadas (dependientes, profesores, independientes, TENS, mecánicos).

Decisión: campañas emocionales y territoriales > campañas premium. El valor es el sentido de pertenencia.
05 · Edad estimada desde RUN

Una estimación honesta, no un invento

⚠ Advertencia metodológica (importante). El RUN chileno se entrega de forma correlativa: a mayor número, más reciente la inscripción. Existe una correlación estadística entre el número de RUN y el año de nacimiento, pero NO permite conocer la edad exacta (hay inscripciones tardías, extranjeros nacionalizados y variación natural). Por eso no mostramos una edad exacta: mostramos un tramo etario aproximado y marcamos el nivel de confianza como bajo. Úsese como orientación de mezcla generacional, nunca como dato individual definitivo.

Tramo etario estimado

Qué se puede concluir (con prudencia)

Solo a nivel de grupo, no de persona

La mezcla sugiere un núcleo adulto-joven y adulto (26 a 54 años), que concentra ~82% de la base estimada. Coherente con socios que sostienen económicamente el club.

Hay poca presencia estimada de menores de 25, lo que abre una oportunidad de captación juvenil (universitarios, hinchada joven).

Alternativa recomendada: para segmentar por edad de verdad, agregar una pregunta de fecha de nacimiento en el próximo formulario. Es el único método robusto.

09 · Evaluación predictiva

¿Esta base permite predecir? Respuesta honesta: todavía no

¿Predicción robusta?

No. Falta lo esencial: no hay una variable objetivo (ej. "renovó / no renovó") ni historial de comportamiento.

¿Variable objetivo?

No existe. La base es declarativa (datos de contacto de un formulario), no transaccional.

¿Qué sí se puede?

Un scoring semipredictivo por reglas (sección 07) para priorizar contacto. Es lo correcto y transparente.

El camino a un modelo real: ejecutar la estrategia "Data Pirata" (registrar asistencia, compras y renovaciones). Con 1–2 temporadas de historial se puede entrenar un modelo de churn (quién dejará de ser socio) y de propensión (quién comprará). No forzamos hoy un modelo que la base no soporta.

11 · Metodología

Cómo se hizo, paso a paso

Flujo del pipeline (Python)

  1. Lectura de la hoja BD_CORPO (325 × 12).
  2. Renombrado a nombres técnicos por posición.
  3. Reparación de teléfonos (notación científica → +569).
  4. Limpieza de RUN + validación de dígito verificador (módulo 11).
  5. Normalización de emails (minúsculas, sin espacios).
  6. Normalización de ciudad (corrección de typos).
  7. Detección de duplicados (RUN / email / móvil / nombre+ciudad).
  8. Estimación de tramo etario desde RUN (con nivel de confianza).
  9. Scoring semipredictivo + segmentación.
  10. Preparación de geocodificación (geopy/Nominatim).
  11. Exportación de CSV limpio + JSON de agregados.

Decisiones y supuestos clave

• Edad: no se afirma edad exacta desde RUN; solo tramo con confianza baja.

• Duplicados: se conserva el primero y se marca el resto para validación, no se borra automáticamente.

• Móviles: 9 dígitos que parten en 9 → móvil; se rescata notación científica.

• Consentimiento: "Si/ok/okey" cuentan como aceptación explícita (solo 8 registros).

• Reproducible: el mismo script corre sobre cualquier club de la red cambiando el archivo fuente.

12 · Integración

Cómo poner esto a correr

Paso a paso

  1. Instalar dependencias:
    pip install pandas numpy openpyxl geopy
  2. Dejar BD_CAPTA_CORPOCU.xlsx junto al script.
  3. Correr limpieza: python pipeline_corpocu.py
  4. Con geocodificación real: python pipeline_corpocu.py --geo
  5. Revisar carpeta salidas_corpocu/.
  6. Abrir dashboard_corpocu.html en el navegador.
